Designator PDU

Size: 704
Description: Designating operations shall be communicated by issuing a Designator PDU. The Designator PDU shall contain the following fields:

a) PDU Header - This field shall contain data common to all DIS PDUs. The PDU Header shall be represented by the PDU Header Record (see 5.3.24).
b) Designating Entity ID - This field shall identify the entity that is positioning the designator. This field shall be represented by an Entitiy Identifier Record (see 5.3.14).
c) Code Name - This field shall identify the code name for the designator system. This field shall be represented by a 16-bit enumeration (see Section 9 in EBV-DOC).
d) Designated Entity ID - This field shall identify the entity that is currently being designated. This field shall contain D-SPOT_NO_ENTITY if the designator spot is not on an entity. This field shall be represented by an Entity Identifier Record (see 5.3.14).
e) Designator Code - This field shall identify the designator code being used by the designating entity. This field shall be represented by a 16-bit enumeration (see Section 8 in EBV-DOC).
f) Designator Power - This field shall identify the designator output power in watts. This field shall be represented by a 32-bit floating point number.
g) Designator Wavelength - This field shall identify the designator wavelength in units of microns. This field shall be represented by a 32-bit floating point number.
h) Designator Spot with respect to Designated Entity - This field shall specify the location of the designator spot with respect to the designated entity's coordinate system when the spot is on an entity. If the Designated Entity is D-SPOT_NO_ENTITY, then this field shall contain the location [0.0, 0.0]. This field shall be represented by an Entity Coordinate Vector Record (see 5.3.33.1).
i) Designator Spot Location - This field shall identify the location of the designator spot with respect to the world coordinate system. This field shall be represented by a World Coordinates Record (see 5.3.34).
j) Dead Reckoning Parameters - This field will be used to provide parameters for dead reckoning the position of the designator spot. Dead Reckoning algorithm in use, and Entity Acceleration shall be included as part of the dead reckoning parameters.

1) Dead Reckoning Algorithm - This field shall specify the dead
reckoning algorithm in use by the issuing entity. This field shall
be represented by an 8-bit enumeration. Values for this
enumerations are defined in Section 4 of EBV-DOC.

2) Entity Linear Acceleration - This field shall specify the designator
spot's linear acceleration. This field shall be represented by a
Linear Acceleration Vector Record (see 5.3.33.2).
